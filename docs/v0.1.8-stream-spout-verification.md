# v0.1.8 Stream Camera(Spout)実機確認手順

## 事前準備

- VRChatでOSCを有効にする。
- 通常版RCの `ClipForVRChat-v0.1.8-rcN-windows-amd64.exe` を起動する。
- 分離版の確認も行う場合は、`ClipForVRChat-v0.1.8-rcN-windows-amd64-separated.zip` を展開したフォルダに `ClipForVRChat.exe`、`spout-capture.exe`、`SpoutLibrary.dll` があることを確認する。
- ClipForVRChatの設定画面で「自動撮影」タブを開く。
- 撮影方式を `Stream` にする。
- `helper確認` を押し、内蔵helperから展開された `spout-capture.exe` のパスとversionが表示されることを確認する。分離版で失敗する場合は `SpoutLibrary.dll` が同じフォルダに残っているか確認する。
- VRChatでStream Cameraを起動し、`sender一覧更新` を押す。
- senderが複数ある場合はVRChat/Stream Cameraに該当するsenderを選ぶ。判断不能な場合はログとsender一覧を保存する。

## 手動Camera表示中の確認

- VRChat内でStream Cameraを表示したまま、正面、背後、斜めの各構図で「テスト撮影」を押す。
- 設定画面のテスト結果に保存先、sidecar候補、Discord URLが表示されることを確認する。
- 保存画像がデスクトップ全体、VRChatウィンドウ周辺、白一色、黒一色、透明画像ではなく、Stream Camera映像そのものになっていることを確認する。
- 画像と同じ場所の `.json` を開き、`stream.backend=spout`、`stream.sender_name`、`vrchat.world_id`、`vrchat.instance_id`、`users`、`files.sha256` が記録されていることを確認する。

## Cameraを閉じた状態からの確認

- VRChatのCameraを閉じた状態にする。
- 自動撮影スケジュールまたは構図の「テスト撮影」を実行する。
- ClipForVRChatがOSCでStream Cameraを起動し、Spoutフレーム取得後に設定どおり閉じることを確認する。
- 失敗した場合は、設定画面のエラー、`logs/YYYY-MM-DD.log` の `spout capture`、`osc send`、`stream start wait` を確認する。

## 失敗時に提出するもの

- `logs/YYYY-MM-DD.log`
- `config.json`
- `history.json`
- 保存された画像と同名の `.json`
- 問題の画像
- 自動撮影タブのsender一覧が分かるスクリーンショット

Discord添付画像内のメタデータ保持はDiscord側仕様として保証されない。Discord本文とsidecar JSONを正本として確認する。
