# v0.1.8 埋め込みメタデータ確認手順

## 自動テストで確認する範囲

`go test ./...` で以下を確認する。

- PNGの `iTXt` と `eXIf` に自動撮影メタデータを書き込める。
- JPEGのEXIF APP1に自動撮影メタデータを書き込める。
- PNG/JPEGどちらも `ReadAutoCaptureEmbeddedMetadata(path)` で読み戻せる。
- 同じ画像へ再書き込みしても、自分のメタデータを重複させず差し替える。
- ユーザー数が多い場合はpayload上限内に切り詰め、`users_truncated` と `metadata_warnings` を残す。

## 実機確認

- 自動撮影タブで「画像埋め込みメタデータ」をONにする。
- ユーザーIDを画像内に入れる必要がある場合だけ「ユーザーIDを画像に埋め込む」をONにする。
- PNG/JPEGそれぞれでテスト撮影する。
- 画像と同じ場所のsidecar JSONで `users`、`vrchat.world_id`、`vrchat.instance_id`、`files.sha256` を確認する。
- 任意で `exiftool <image>` を使い、ImageDescriptionまたはPNG eXIf/iTXtにClipForVRChatメタデータがあることを確認する。

## Discord投稿後の扱い

Discord添付後に画像メタデータが保持されるかは保証しない。
確認時は、ローカル画像とsidecar JSONを正本にし、Discord本文に同席ユーザー情報が含まれることを確認する。
Discordからダウンロードした画像にメタデータが残っていない場合でも、sidecarと本文が残っていればv0.1.8の設計上は正常と扱う。

## 失敗時に提出するもの

- `config.json`
- `logs/YYYY-MM-DD.log`
- ローカル保存画像
- 画像と同名の `.json`
- Discord投稿URL
- `exiftool` を使った場合はその出力
