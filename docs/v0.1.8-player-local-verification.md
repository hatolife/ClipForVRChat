# v0.1.8 player_local構図実機確認手順

## 仕様前提

`player_local` は標準OSCだけでプレイヤーrootを自動取得しない。
v0.1.8では、手動で保存した「プレイヤー基準Pose」を基準に、各構図のローカル位置/回転をworld poseへ変換する。
プレイヤーが大きく移動/回転した後は、必要に応じて基準Poseを保存し直す。

## 基準Poseの保存

- VRChatでUser Cameraを表示する。
- 顔/目線付近など、構図の基準にしたい位置へCameraを置く。
- 自動撮影タブで「現在Poseをプレイヤー基準に保存」を押す。
- `プレイヤー基準Pose` が保存済み表示になることを確認する。

## 構図確認

- 正面: 顔が中心に入り、プレイヤー正面側から撮影される。
- 背後: 三人称視点のゲームのように、後頭部側から少し見下ろす。
- 斜め: 正面側かつ少し横側から、少し見下ろす。

各構図で次を確認する。

- 「このPoseへカメラ移動」で期待位置へ移動する。
- 「テスト撮影」で保存画像とsidecarが作成される。
- sidecarの `view.coordinateSpace=player_local` と `resolved_pose` が記録される。
- Desktop/VR、アバター身長差、座り状態でずれが大きい場合は、基準Poseを保存し直してから再確認する。

## 失敗時に提出するもの

- `config.json`
- `logs/YYYY-MM-DD.log`
- 撮影画像と同名の `.json`
- 基準Poseを保存した時刻
- 期待と違う構図が分かる画像
