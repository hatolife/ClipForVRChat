# ClipForVRChat 仕様

## 概要

ClipForVRChat は、VRChat ユーザー向けの Windows デスクトップアプリです。

VRChat の外部画像読み込みで扱いやすいように、画像を 2048x2048 以内へ縮小し、必要に応じて Discord に投稿し、投稿された画像URLをすばやくクリップボードへコピーします。

アプリ本体は Go と Wails で実装します。

## 目的

- 入力画像を 2048x2048 以内に収める。
- 縦横比を維持する。
- 元画像が制限内なら拡大しない。
- 設定に応じて縮小画像をローカル保存する。
- 設定に応じて縮小画像を Discord Webhook に投稿する。
- 設定に応じて画像内のQRコードURLを検出し、Discord投稿本文と結果画面に表示する。
- 可能な場合、画像URLまたは画像データをクリップボードへ保持する。
- 複数画像のときはURL一覧UIを表示する。
- 設定は exe と同じ場所の JSON に保存する。
- 取得した画像URLを履歴として保存し、起動時に過去画像を確認できるようにする。

## やらないこと

- ユーザーがJSONを直接編集する前提の設定運用。
- 初期版でのDiscord以外のアップロード先対応。
- Discord添付URLの長期永続性保証。

## 入力

### 引数なし

引数なしで起動された場合、現在のクリップボード画像を処理します。

クリップボードに画像がない場合は、画像をコピーしてから再実行するか、画像ファイルを引数に指定するよう案内するエラーを表示します。

### 画像ファイル引数

1枚以上の画像ファイルが引数に指定された場合、それらを順番に処理します。

初期対応形式:

- JPEG
- PNG
- WebP

対応していないファイルは、ユーザーが次に何をすべきかわかるエラーとして表示します。

### 設定ファイル引数

`.json` 設定ファイルが1つだけ指定された場合、設定画面を開きます。

画像ファイルと設定ファイルが混在している場合は、設定編集と画像処理は別々に起動するよう案内するエラーを表示します。

### CLIオプション

以下のオプションは起動ロック取得やGUI起動より前に処理します。

- `--version`: アプリ名とバージョンを表示して終了する。
- `--help`, `-h`: 対応しているCLIオプションを表示して終了する。

Windows配布exeは通常のGUIアプリとしてビルドするため、通常起動時にコンソールを表示しません。GUIサブシステムexeの仕様により、PowerShell、cmd、Git BashではCLI出力が次のプロンプト表示後に出る場合があります。

cmdなどのコードページ差で日本語ヘルプが文字化けしないよう、Windowsではコンソール出力に `WriteConsoleW` を使います。

### 初回起動

設定ファイルが存在しない状態で通常起動した場合は、最初に設定画面を表示します。

ユーザーが設定を保存したあと、起動時の入力を使って通常処理を続行します。

例:

- 引数なしで初回起動: 設定保存後、クリップボード画像を処理する。
- 画像ファイルを指定して初回起動: 設定保存後、その画像ファイルを処理する。

## 画像処理

### 縮小ルール

画像の横幅と高さがどちらも以下に収まるように縮小します。

```txt
2048x2048
```

縦横比は維持します。

すでに制限内の画像は拡大しません。

### 出力形式

出力形式は設定で選択できます。

初期値:

```txt
PNG
```

選択肢:

- PNG
- JPG

入力形式には依存せず、設定された出力形式で保存・投稿します。

### JPEG品質

JPEG品質は設定可能です。

初期値:

```txt
92
```

JPEG品質はJPG出力時だけ使用します。PNG出力時の設定画面では、JPEG品質が不要であることを表示します。

## ローカル出力

ローカル保存は設定でON/OFFできます。

ONの場合、縮小画像をファイルとして保存します。

### 出力先

出力先フォルダは設定可能です。

初期値:

```txt
./output
```

相対パスの場合は exe と同じフォルダを基準にします。

出力先フォルダ入力欄に `"` で囲われた文字列が貼り付けられた場合、保存時に前後の `"` を取り除きます。

### 出力名

初期サフィックス:

```txt
_2048
```

例:

```txt
image.png -> image_2048.png
image.jpg -> image_2048.jpg
```

初期設定では既存ファイルを上書きしません。

出力先がすでに存在する場合は連番を付与します。

```txt
image_2048.png
image_2048_2.png
image_2048_3.png
```

クリップボード画像の場合、保存ファイル名には処理時刻を含めます。

例:

```txt
clipboard_20260621_133132_2048.png
```

## 履歴

Discord投稿で取得した画像URLは履歴として保存します。

履歴ファイル:

```txt
history.json
```

配置場所:

- 設定ファイルと同じフォルダ。

履歴に保存する主な内容:

- URL
- サムネイル
- 元ファイル名
- ローカル出力先
- DiscordメッセージID
- Webhook ID/token
- 作成日時
- 結果表示からクリア済みかどうか
- Discord上で削除済みかどうか

結果画面のクリアボタンで表示を消した場合も、履歴からは削除しません。該当履歴にはクリア済みフラグを立てます。

アプリ起動時、現在の処理結果がない場合は、未クリアの履歴画像を結果欄に表示します。

## Discord投稿

Discord投稿には Discord Webhook を使います。

縮小済み画像を、設定されたWebhookへ投稿します。

投稿後、Webhookレスポンスから添付画像の直リンクURLを取得します。

QRコードURL検出がONの場合、画像内のQRコードを読み取ります。

- QRコードにURLが含まれている場合、Discord投稿本文にURL一覧を記載します。
- 1枚の画像に複数のQRコードURLがある場合、読み取れたURLをすべて記載します。
- QRコードにURL以外の文字列が含まれる場合、Discord投稿本文と結果画面には表示しません。
- QRコードが見つからない場合もエラーにはしません。

Discord投稿がONでWebhook URLが未設定または不正な場合は、設定画面でWebhook URLを設定するよう案内するエラーを表示します。

Discord上の画像削除は、Webhook投稿レスポンスから取得したメッセージIDを使ってWebhookメッセージを削除します。

古い履歴など、削除に必要なメッセージIDまたはWebhook情報がない場合は、Discord APIによる削除はできません。

## VRChat写真自動投稿

VRChat写真自動投稿をONにすると、指定した写真フォルダを監視します。

初期写真フォルダ:

```txt
%USERPROFILE%\Pictures\VRChat
```

起動後に新しく保存された対応画像を検知した場合、通常の画像処理と同じように縮小、ローカル保存、Discord投稿、履歴保存を行います。

対応形式:

- JPEG
- PNG
- WebP

監視対象は指定フォルダとそのサブフォルダです。

自動投稿専用Webhook URLを設定できます。

- 設定されている場合: 自動投稿専用Webhook URLへ投稿する。
- 未設定の場合: 通常のDiscord Webhook URLへ投稿する。

初期対応ではOSCイベントではなく、写真フォルダの定期スキャンで検知します。実際に保存された画像を確実に処理するためです。

スキャン間隔は設定画面で変更できます。初期値は2秒です。

## スクリーンショット自動投稿

スクリーンショット自動投稿をONにすると、指定したScreenshotsフォルダを監視します。初期値はOFFです。

初期Screenshotsフォルダ:

```txt
%USERPROFILE%\Pictures\Screenshots
```

起動後に新しく保存された対応画像を検知した場合、通常の画像処理と同じように縮小、ローカル保存、Discord投稿、履歴保存を行います。

スクリーンショット自動投稿専用Webhook URLを設定できます。

- 設定されている場合: スクリーンショット自動投稿専用Webhook URLへ投稿する。
- 未設定の場合: 通常のDiscord Webhook URLへ投稿する。

スキャン間隔はVRChat写真自動投稿とは別に設定できます。初期値は2秒です。

## クリップボード出力

### 1枚の場合

画像が1枚だけでDiscord投稿に成功した場合、設定がONなら投稿された画像URLを自動でクリップボードへコピーします。

ローカル保存とDiscord投稿が両方OFFの場合は、縮小後の画像データをクリップボードへ保持します。

### 複数枚の場合

複数画像を処理し、Discord投稿に成功した場合はURL一覧UIを表示します。

一覧要素をクリックすると、そのURLをクリップボードへコピーします。

ローカル保存とDiscord投稿が両方OFFで複数画像が指定された場合は、複数画像を直接クリップボードへ保持できないため、ローカル保存またはDiscord投稿をONにするか、1枚ずつ処理するよう案内するエラーを表示します。

## UI

UIは Wails で実装します。

### 全体レイアウト

ウィンドウ上部には常にヘッダーを表示します。

ヘッダー左側:

- アプリ名

ヘッダー右側:

- 使い方ボタン
- 情報ボタン
- 設定ボタン

使い方、情報、設定ボタンは通常画面、結果画面、エラー画面のどの状態でも押せるようにします。

各ボタンは初期画面では白で表示し、対応する画面を表示中は選択色で表示します。選択中のボタンを再度押すと初期画面へ戻ります。

バージョン情報はタイトルバーと情報画面に表示し、ヘッダーには表示しません。

### 通常画面

通常画面は、画像処理の入口として機能する画面です。

左側または上側に大きなドロップ領域を表示します。

表示内容:

- 画像をドラッグ&ドロップできること
- 複数画像をまとめて処理できること
- `config.json` をドロップすると設定画面を開くこと
- クリップボード画像を処理するボタン

右側または下側に結果領域を表示します。

結果がない場合は「まだ処理結果はありません。」のように表示します。

引数なしで通常起動した場合は、クリップボード処理を自動実行せず、通常の初期画面を表示します。

結果領域の右上にはクリアボタンを表示します。クリアボタンを押すと、現在の結果表示を消します。

クリアボタンの横には履歴ボタンを表示します。履歴ボタンを押すと、画像履歴画面へ遷移します。

### 使い方画面

使い方ボタンを押すと、アプリの基本操作を説明する画面を表示します。

説明内容:

- 画像ファイルをドラッグ&ドロップして処理できること。
- 複数画像をまとめて処理できること。
- クリップボード画像を処理できること。
- 設定画面でローカル保存、Discord投稿、出力形式、出力先などを変更できること。
- Discord投稿にはWebhook URLが必要なこと。
- 結果サムネイルをクリックすると画像URLを再コピーできること。

使い方画面にはDiscord公式のWebhook説明ページへのリンクを表示します。

```txt
https://support.discord.com/hc/ja/articles/228383668-%E3%82%A6%E3%82%A7%E3%83%96%E3%83%95%E3%83%83%E3%82%AF%E3%81%AE%E3%81%94%E7%B4%B9%E4%BB%8B
```

リンクをクリックすると、既定のブラウザで開きます。

### URL一覧

URL一覧は以下の場合に表示します。

- 画像処理後にUIを表示する設定になっている場合。
- 複数画像を処理した場合。

結果表示はサムネイル画像中心にします。

- サムネイルをクリックすると、その画像URLをクリップボードへ再コピーします。
- 結果領域には「サムネイルをクリックすると画像URLをコピーできる」ことを明記します。
- URL文字列は主表示にしません。
- 結果が大量にある場合、スクロールは結果領域だけに発生させ、ドロップ領域やヘッダーは伸ばしません。
- サムネイル上にマウスを置くと、半透明オーバーレイで「クリックでURLをコピー」と表示します。

複数画像が入力された場合は、入力直後に全ファイル分の結果枠を表示します。

- 処理中の枠には処理中であることを表示します。
- 可能な範囲で円環状の進捗表示を出します。
- 各画像の処理完了後、その枠を結果サムネイルへ置き換えます。
- 結果領域上部に全体の処理件数と進捗バーを表示します。
- QRコードURL検出がONの場合、各結果サムネイル内に検出したURL一覧を表示します。
- 結果サムネイル内のQRコードURLをクリックすると、そのURLをクリップボードへコピーします。

### 設定画面

設定画面は、用途ごとに以下のまとまりで表示します。

- 出力: ローカル保存、Discord投稿、1枚時のURL自動コピー、QRコードURL検出、出力形式、JPEG品質、UI表示。
- 保存: 出力先フォルダ、ファイル名サフィックス。
- Discord: Webhook URL。
- VRChat写真自動投稿: ON/OFF、スキャン間隔、写真フォルダ、自動投稿用Webhook URL。
- スクリーンショット自動投稿: ON/OFF、スキャン間隔、Screenshotsフォルダ、スクリーンショット投稿用Webhook URL。

各設定項目は、左側に設定名と説明、右側に入力欄または操作部品を表示します。

ON/OFF項目はiOS風のトグルスイッチで表示します。

出力先フォルダにはフォルダ選択ダイアログを開くボタンを表示します。

サフィックス欄には保存名の例を表示します。

Discord Webhook URL欄には、設定方法の説明とDiscord公式ヘルプへのリンクを表示します。

設定画面下部には以下の操作を表示します。

- 保存
- 閉じる

保存が成功した場合、設定画面を閉じて通常画面へ戻ります。

閉じるを押した場合、設定は保存せず通常画面へ戻ります。

### 情報画面

情報画面には以下を表示します。

- プログラム名
- バージョン
- GitHub URL

GitHub URLをクリックすると、既定のブラウザでリポジトリを開きます。

情報画面にはOSSライセンス画面へ遷移するボタンを表示します。

### OSSライセンス画面

OSSライセンス画面には、アプリで使用しているOSSの一覧を表示します。

表示内容:

- OSS名
- ライセンス種別
- 著作権表記
- プロジェクトURL

ClipForVRChat本体のライセンス表記もこの一覧に含めます。

URLコピー後、UI内に短いトーストを表示します。

```txt
コピーしました
```

### 画像履歴画面

画像履歴画面には、削除済み・クリア済みを含めたすべての履歴画像を表示します。

履歴画像は複数選択できます。

- Ctrlクリック: 個別に選択または選択解除。
- Shiftクリック: 直前の選択位置から範囲選択。

履歴画面上部には以下のボタンを表示します。

- 選択をDiscordから削除
- 削除済みURLの履歴を削除
- 閉じる

「選択をDiscordから削除」は、選択した履歴のDiscord投稿メッセージを削除します。履歴情報や元画像データは削除しません。

「削除済みURLの履歴を削除」は、履歴内のURLへアクセスできるか確認し、既に参照できないURLの履歴だけを履歴ファイルから削除します。

### 設定ボタン

通常の結果画面やエラー画面にも設定ボタンを表示します。

設定ボタンを押すと、現在使用中の設定ファイルを編集する設定画面を表示します。

### ファイルドラッグ&ドロップ

起動済みウィンドウへのファイルドラッグ&ドロップに対応します。

`.json` 設定ファイルを1つだけドロップした場合:

- その設定ファイルを対象に設定画面を表示する。

画像ファイルを1枚以上ドロップした場合:

- 入力ファイルとして通常処理を実行する。
- 複数画像のドロップにも対応する。

画像ファイルと設定ファイルが混在している場合:

- 処理せず、設定編集と画像処理を分けるよう案内するエラーを表示する。

ドロップ対象はアプリウィンドウ全体とします。

ドラッグ中は半透明オーバーレイを表示し、ドロップ可能であることを案内します。

## 設定

設定はJSONで保存します。

初期配置:

```txt
config.json
```

ファイルは exe と同じ場所に置きます。

通常ユーザーは設定UIから編集します。JSONはデバッグしやすいよう、読みやすく安定した構造にします。

### 設定例

```json
{
  "image": {
    "maxWidth": 2048,
    "maxHeight": 2048,
    "suffix": "_2048",
    "overwrite": false,
    "jpegQuality": 92,
    "outputDirectory": ""
  },
  "output": {
    "saveLocal": true,
    "uploadDiscord": true,
    "showUi": "auto",
    "copySingleUrlToClipboard": true,
    "detectQrCodeUrls": false
  },
  "discord": {
    "webhookUrl": ""
  },
  "autoPhoto": {
    "enabled": false,
    "photoDirectory": "%USERPROFILE%\\Pictures\\VRChat",
    "webhookUrl": "",
    "scanIntervalSeconds": 2
  },
  "screenshotAutoPost": {
    "enabled": false,
    "screenshotDirectory": "%USERPROFILE%\\Pictures\\Screenshots",
    "webhookUrl": "",
    "scanIntervalSeconds": 2
  }
}
```

### UI表示モード

設定値:

- `auto`
- `always`
- `never`

`auto` の挙動:

- 1枚の投稿URLコピー成功時: UIを表示しない。
- 複数の投稿URLがある場合: URL一覧UIを表示する。
- エラー時: UIまたはダイアログで案内する。

## エラーメッセージ方針

エラーは、何が起きたかだけでなく、ユーザーが次に何をすべきかも含めます。

例:

- クリップボードに画像がない: `クリップボードに画像がありません。画像をコピーしてから再実行するか、画像ファイルを exe にドラッグしてください。`
- 複数画像をクリップボードへ直接出力できない: `複数画像はクリップボードへ直接保持できません。ローカル保存またはDiscord投稿をONにするか、1枚ずつ処理してください。`
- Webhook URL未設定: `Discord投稿がONですがWebhook URLが未設定です。設定画面でWebhook URLを設定してください。`

## Release

GitHub Release のWindows配布zipには以下を含めます。

- `ClipForVRChat.exe`
- `Release-signing-public-key.url`
- `README.md`
- `LICENSE`

Release asset には配布zipに加えて以下を添付します。

- `ClipForVRChat-vX.Y.Z-windows-amd64.zip.sha256`
- `ClipForVRChat-vX.Y.Z-windows-amd64.exe.asc`

zipファイル名にはバージョンを含めます。

例:

```txt
ClipForVRChat-v0.4.0-windows-amd64.zip
```
