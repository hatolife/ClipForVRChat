# ClipForVRChat 設定画面仕様

## 目的

設定画面は、通常ユーザーが `config.json` を直接編集せずに必要な動作を切り替えられるようにする画面です。

設定項目は、ユーザーが判断しやすい粒度でカテゴリ分けします。頻繁に触る可能性が低い詳細値は、原則として設定画面に出しません。

## 画面構成

- 見出し
  - 左側に `設定` を表示する。
  - 右側に `保存` と `閉じる` ボタンを表示する。
  - 保存が成功した場合は `保存しました` を短く表示する。
- カテゴリタブ
  - 見出しの下に表示する。
  - タブはカテゴリ切り替え専用にする。
  - 選択中カテゴリの設定だけを表示する。
- 設定項目
  - 左側に設定名と説明を表示する。
  - 右側に入力欄、トグル、選択欄、ボタンなどの操作部品を表示する。
  - ON/OFF項目はトグルスイッチで表示する。
- 未保存変更
  - 設定画面を離れる前に保存、破棄、キャンセルを選べる確認を表示する。

## カテゴリと項目

### 機能

- VRChat写真自動処理
  - 種別: ON/OFF
  - 説明: VRChat上で撮影されたときに処理する。
- スクリーンショット自動処理
  - 種別: ON/OFF
  - 説明: Win + Shift + S でスクリーンショットが撮られたときに処理する。
- Discord投稿
  - 種別: ON/OFF
  - 説明: 処理した画像をDiscord Webhookへ投稿する。
- QRコードURL検出
  - 種別: ON/OFF
  - 説明: 画像内のQRコードからURLを取得する。

### 処理

- ローカル保存
  - 種別: ON/OFF
  - 説明: 処理した画像をローカルに保存する。
- 投稿URLの自動コピー
  - 種別: ON/OFF
  - 説明: Discordに投稿したURLをクリップボードに保存する。
- 出力先フォルダ
  - 種別: フォルダパス入力 + 選択ボタン
  - 説明: ローカル保存時の保存先。
- ファイル名サフィックス
  - 種別: テキスト入力
  - 説明: ローカル保存時のファイル名末尾に付ける文字。
- 出力形式
  - 種別: 選択
  - 値: `PNG`, `JPG`
  - 説明: 保存または投稿に使う画像形式。
- JPEG品質
  - 種別: 数値入力
  - 説明: JPG出力時の画質。
- 履歴削除時にoutputも削除
  - 種別: ON/OFF
  - 説明: Discord削除済み履歴を削除するとき、ローカル保存したoutput画像も削除する。

### Webhook

- 通常投稿用Webhook URL
  - 種別: パスワード入力
  - 説明: 通常のDiscord投稿先。
- VRChat写真自動処理用Webhook URL
  - 種別: パスワード入力
  - 説明: VRChat写真自動処理だけ別のDiscordチャンネルへ投稿したい場合に使う。空の場合は通常投稿用Webhook URLを使う。
- スクリーンショット自動処理用Webhook URL
  - 種別: パスワード入力
  - 説明: スクリーンショット自動処理だけ別のDiscordチャンネルへ投稿したい場合に使う。空の場合は通常投稿用Webhook URLを使う。
- Webhook作成案内
  - 種別: リンクボタン
  - 説明: Discord公式のWebhook作成案内を開く。

### 更新

- 更新確認
  - 種別: ON/OFF
  - 説明: 起動時にGitHub Releasesを確認し、新しいバージョンがあるか調べる。
- 更新通知
  - 種別: ON/OFF
  - 説明: 新しいバージョンが見つかったとき、画面上部に通知を表示する。

## 画面に出さない設定

- 最大入力サイズ
  - 通常ユーザーが頻繁に調整する値ではないため、設定画面には表示しない。
  - 内部設定値としては維持する。
- UI表示
  - 通常利用では変更頻度が低いため、設定画面には表示しない。
  - 内部設定値としては維持する。

## 依存関係と無効化

- Discord投稿がOFFの場合
  - 投稿URLの自動コピーをグレーアウトし、編集不可にする。
  - Webhookカテゴリの入力項目をグレーアウトし、編集不可にする。
  - VRChat写真自動処理をグレーアウトし、編集不可にする。
  - スクリーンショット自動処理をグレーアウトし、編集不可にする。
- VRChat写真自動処理がOFFの場合
  - VRChat写真自動処理用Webhook URLをグレーアウトし、編集不可にする。
- スクリーンショット自動処理がOFFの場合
  - スクリーンショット自動処理用Webhook URLをグレーアウトし、編集不可にする。
- ローカル保存がOFFの場合
  - 出力先フォルダをグレーアウトし、編集不可にする。
  - ファイル名サフィックスをグレーアウトし、編集不可にする。
  - 履歴削除時にoutputも削除をグレーアウトし、編集不可にする。
- 出力形式がPNGの場合
  - JPEG品質をグレーアウトし、編集不可にする。
- 更新確認がOFFの場合
  - 更新通知をグレーアウトし、編集不可にする。

## 更新通知

更新確認がONの場合、起動時にGitHub Releasesの最新Releaseを確認します。

新しいバージョンがある場合は、ヘッダー下に更新通知を表示します。

更新通知には以下を表示します。

- 新しいバージョン番号
- GitHub Releasesを開くボタン
- BOOTHを開くボタン
- 通知を閉じる×ボタン

更新通知の×ボタンを押した場合、その起動中は通知を再表示しません。

更新確認がOFFの場合、起動時にGitHub Releasesの確認を行いません。

更新通知がOFFの場合、新しいバージョンが見つかっても画面上部の通知を表示しません。
