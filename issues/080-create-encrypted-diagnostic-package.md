# 080 暗号化診断パッケージ作成

## 問題

ユーザー環境で不具合が発生したとき、ログ、設定、履歴、実行ファイルをまとめて確認する手段がない。
また、これらにはファイルパスやWebhook設定の有無などの利用環境情報が含まれるため、GitHub Issueなどに添付する場合は復号なしに中身やファイル一覧が見えない形にする必要がある。

## 期待する挙動

- 起動時に実行ファイルのバージョン、SHA256、読み込んだ設定サマリを診断ログへ記録する。
- 情報画面から、ログ、`history.json`、`config.json`、起動中のexeを含む診断パッケージを作成できる。
- 診断パッケージは公開鍵暗号で保護され、復号前にファイル内容や一覧が分からない。
- ユーザー確認用に暗号化前zipを残し、添付用には暗号化済みの `.zip.gpg` を作成する。
- 暗号化に使う公開鍵が暗号化用途に対応していない場合は、ユーザーに分かるエラーを表示する。

## 受け入れ条件

- 起動ログにバージョン、exe SHA256、設定サマリが出力される。
- Webhook URLなどの秘密値はログへ平文出力されない。
- 診断パッケージ作成時に確認用zipと添付用 `.zip.gpg` が作成される。
- 暗号化用公開鍵が暗号化用途でない場合、その旨のエラーになる。
