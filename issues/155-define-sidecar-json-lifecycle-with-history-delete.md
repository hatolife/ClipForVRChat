# sidecar JSONの履歴・削除ライフサイクルを定義する

## 問題

自動撮影では画像と同じ場所に `画像ファイル名 + ".json"` のsidecar JSONを保存する。
一方、履歴にはsidecar JSONのパスが保存されず、ローカル削除や履歴削除は画像ファイルだけを対象にする。
その結果、画像削除後にsidecar JSONだけが残る、または履歴からsidecarを辿れない可能性がある。

## 期待する挙動

sidecar JSONを画像の付随メタデータとして扱うのか、独立した監査ログとして残すのかを仕様化する。
仕様に従って、履歴表示、ローカル削除、履歴削除、診断データで一貫した扱いにする。

## 受け入れ条件

- sidecar JSONを画像削除時に一緒に消すか、保持するかを仕様で決める。
- 一緒に消す場合は、履歴にsidecarパスを保存し、削除対象を安全に検証する。
- 保持する場合は、履歴/README/設定UIで「画像削除後もメタデータが残る」ことを説明する。
- 履歴画面からsidecar JSONの存在状態を確認できるかを決める。
- ローカル削除、履歴削除、Discord削除済みpurgeの単体テストを追加する。
- 診断データ生成時にsidecar JSONを含めるか、パス一覧だけにするかを決める。
