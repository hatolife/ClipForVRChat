# config読み込み失敗時にアクティブ設定パスを変更しない

## 問題

不正な `config.json` をドロップまたは引数指定した場合、読み込みに失敗しても内部の設定パスがそのファイルへ切り替わり、保存先や履歴・診断ログの場所が意図せず変わる可能性がある。

## 期待する挙動

設定ファイルの読み込みに成功した場合だけ、アクティブな設定パスを変更する。読み込み失敗時は既存の設定パスを維持する。

## 受け入れ条件

- GUIで不正なconfigを開いても `configPath` が変更されない。
- 起動引数で不正なconfigを指定した場合も、既定の設定パスを維持してエラー表示する。
- 成功時の外部config編集導線は維持される。
