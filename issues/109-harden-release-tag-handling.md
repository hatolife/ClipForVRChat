# 109 Release workflow のタグ名取り扱いを安全化する

## 問題

Release workflow で `github.ref_name` を PowerShell / Bash の `run` コマンドや成果物ファイル名へ直接埋め込んでいる箇所があり、細工された `v*` タグ名により CI 上でコマンド注入が発生する恐れがある。

## 期待する挙動

タグ名は workflow 内で検証し、シェルコマンドへは環境変数経由で渡す。成果物ファイル名には安全な文字だけを使い、既存の正式リリース・RC・検証用 draft リリース運用を維持する。

## 受け入れ条件

- `run` コマンド内で `github.ref_name` を直接展開しない。
- タグ名から生成する成果物名は安全な形式へ制限またはサニタイズされる。
- 正式リリース `vX.Y.Z`、RC `vX.Y.Z-rcW`、検証用 `v*` draft リリースの判定が維持される。
