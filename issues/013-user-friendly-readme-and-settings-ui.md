# ユーザーフレンドリーなREADMEと設定画面改善

## 問題

READMEがコマンドライン起動寄りで、GUIアプリとして使うユーザーにわかりにくい。初期画面も、単に起動しただけでクリップボード画像エラーが出るため困惑しやすい。設定画面も項目の意味や設定方法が伝わりにくい。

## 期待する挙動

- READMEをGUI操作中心に書き直す。
- PCに詳しくないユーザーでも、ダウンロード、初回設定、画像処理、Discord設定がわかる文章にする。
- 配布zipの説明は使い方に統合し、最新リリースURLを記載する。
- Discord設定は公式ドキュメントリンク付きで丁寧に説明する。
- 引数なし起動ではクリップボード処理を自動実行せず、通常の初期画面を表示する。
- 出力先フォルダの初期値は exe 基準の `./output` にする。
- 出力先フォルダ入力欄は前後の `"` を自動で外す。
- 出力先フォルダ選択ダイアログを開くボタンを追加する。
- サフィックスには保存名の例を表示する。
- Discord Webhook URL欄に設定方法の説明と公式ドキュメントリンクを表示する。
- 使い方画面のDiscord公式リンクをWebhook作成位置付きURLへ更新する。
- 複数画像処理中は全体の進捗を表示する。
- クリアボタン長押しは3秒にし、長押し中は左から右へ進む進捗バー風アニメーションを表示する。
- クリアボタンにマウスオーバーしたとき、3秒長押しで履歴画面へ遷移することを表示する。
- ClipForVRChat本体のライセンス表記はOSSライセンス画面ではなく情報画面に表示し、著作権者は hatolife とする。
- 情報画面にバグ報告先としてGitHub Issuesへの案内を表示する。
- 設定画面は左に項目名と説明、右に入力欄の形式へ統一する。
- ON/OFF項目はiOS風トグルスイッチにする。

## 受け入れ条件

- 引数なし起動で「クリップボードに画像がありません」のエラーが出ない。
- READMEの使い方がGUI操作中心になっている。
- 設定画面の各項目に説明文があり、入力欄が右側に揃っている。
- 出力先フォルダの既定値が `./output` になる。
- `"C:\path"` のような貼り付けでも前後の `"` が保存時に外れる。
- フォルダ選択ボタンで出力先を選べる。
- クリアボタン長押しが3秒で履歴画面に遷移する。
- クリアボタンのホバー説明で、長押しによる履歴画面遷移がわかる。
