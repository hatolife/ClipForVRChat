# アプリの複数起動制限

## 問題

アプリを複数起動できると、`config.json` や `history.json` の同時更新、VRChat写真自動投稿 watcher の重複起動、Discordへの重複投稿が起きる可能性がある。

## 期待する挙動

同じ実行ファイルから起動した ClipForVRChat は、同時に1つだけ動作する。すでに起動中の場合は、新しいプロセスは分かりやすいメッセージを出して終了する。

## 受け入れ条件

- 起動時にアプリ単位のロックを取得する。
- ロック取得に失敗した場合は UI を起動せず終了する。
- 通常終了時にロックを解放する。
- クラッシュ後にロックファイルだけが残っても再起動できる方式にする。
