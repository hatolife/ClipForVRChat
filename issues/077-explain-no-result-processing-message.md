# 077 処理結果がない場合に理由を表示する

## 問題

D&Dや「クリップボード画像を処理」ボタンから明示的に処理したとき、Discord投稿、ローカル保存、QRコードURL取得のいずれも行われない設定だと結果が何も表示されない。現在の文言だけでは、ユーザーが処理に失敗したのか、設定により何も実行されなかったのか分かりにくい。

## 期待する挙動

処理結果がない場合は、設定で有効な出力・取得機能がないため何も表示できる処理が行われなかったことを説明する。

## 受け入れ条件

- Discord投稿、ローカル保存、QRコードURL取得がすべてOFFの場合、処理結果がない理由を表示する。
- QRコードURL取得がONでもQRコードURLが見つからなかった場合、その旨を表示する。
- 明示的に処理したユーザーが、設定を見直せばよいと分かる文言にする。

## 対応

- 処理結果がない場合のメッセージを設定状態に応じて分岐するようにした。
- Discord投稿、ローカル保存、QRコードURL取得がすべてOFFの場合は、表示できる処理結果がない理由を明示するようにした。
- QRコードURL取得だけONでURLが見つからない場合は、URLを取得できなかったことと、Discord投稿またはローカル保存をONにすると画像の処理結果を残せることを表示するようにした。
