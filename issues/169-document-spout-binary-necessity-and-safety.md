# Spout同梱バイナリの必要性と安全性説明を追加する

## 問題

Stream方式で `spout-capture.exe` と `SpoutLibrary.dll` を同梱すると、ユーザーにとって追加の実行ファイル/DLLが増えるため不安要素になる。
現在の説明だけでは、なぜ本体exeに統合しないのか、これらのバイナリが何を行い何を行わないのか、利用者が何を確認すればよいのかが分かりにくい。

## 期待する挙動

READMEや仕様に、Spout同梱バイナリの必要性、安全性の根拠、利用者が確認できる事項を明記する。
安全性は断定的に保証するのではなく、ソース、ビルド、権限、通信、入出力範囲を確認可能な形で説明する。

## 受け入れ条件

- READMEに `spout-capture.exe` と `SpoutLibrary.dll` の必要性を記載する。
- READMEに、ヘルパーの実行範囲、ネットワーク通信を行わないこと、画像保存先以外へ書き込まないことを記載する。
- SPECに配布物としての安全性確認観点を記載する。
- Release Notesにも、追加バイナリへの不安を下げる説明を短く記載する。

## 対応内容

- `README.md` に、Spout同梱バイナリが必要な理由、本体へ統合しない理由、ヘルパーの実行範囲、ネットワーク送信を行わないこと、利用者の確認観点を追記した。
- `src/SPEC.md` に、Stream方式でのヘルパー責務とRelease配布物の安全性確認観点を追記した。
- `RELEASE_NOTES.md` に、`spout-capture.exe` がSpout受信専用の限定されたhelperであることを追記した。
