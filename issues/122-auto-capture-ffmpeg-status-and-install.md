# 自動撮影Stream方式のffmpeg確認と導入導線を追加する

## 問題

v0.1.8-rc9のStream方式では `ffmpeg` を使用するが、設定画面で `ffmpeg` が利用可能か確認できない。
また、`ffmpeg` が未インストールまたはPATH未設定の場合に、原因と導入方法が分かりにくい。

加えて、初期の `ffmpeg` 入力引数がデスクトップ全体を取得する `gdigrab desktop` であるため、Stream Camera取得ではなくデスクトップ全体のスクリーンショットになることが分かりにくい。

## 期待する挙動

- 設定画面から `ffmpeg` の利用可否を確認できる。
- `ffmpeg` が利用できない場合、未インストールまたはPATH未設定であることを表示する。
- 設定画面から `winget install ffmpeg` を実行できる。
- インストール処理の実行結果が画面と診断ログで確認できる。
- デスクトップ全体取得の入力引数は暫定入力であることがUI上で分かる。

## 受け入れ条件

- 自動撮影タブに `ffmpeg` チェックボタンとインストールボタンがある。
- `ffmpeg` がPATHまたは指定パスで見つかる場合、バージョン確認結果が表示される。
- `ffmpeg` が見つからない場合、`ffmpegがインストールされていないかPATHにありません` という趣旨の案内が表示される。
- インストールボタンは `winget install ffmpeg` を実行し、失敗時は原因を表示する。
- `ffmpeg` 実行前にも存在確認し、未検出なら撮影失敗理由として分かる文言を返す。
- 既存の自動撮影、Photo方式、履歴、Discord投稿を壊さない。

## 対応内容

- 設定画面に `ffmpeg確認` と `ffmpegをインストール` ボタンを追加した。
- `CheckFFmpeg` APIでPATHまたは指定パスの `ffmpeg` を解決し、`ffmpeg -version` の結果を表示するようにした。
- `InstallFFmpegWithWinget` APIで `winget install ffmpeg` を実行し、成功/失敗を画面と診断ログへ返すようにした。
- Stream撮影前に `ffmpeg` の存在確認を行い、未検出時は未インストールまたはPATH未設定として表示するようにした。
- `ffmpeg入力引数` の説明で、初期値が確認用のデスクトップ全体取得であることを明示した。
