# 083 不具合報告用データ生成UIを改善する

## 問題

診断データ生成ボタンが情報画面の途中にあり、見つけにくい。
また生成中に他の操作ができてしまい、完了後に生成されたファイルの場所も分かりにくい。

## 期待する挙動

- 情報画面下部の `OSSライセンス` ボタンの右に `不具合報告用データ生成` ボタンを配置する。
- 診断データ作成中は半透明オーバーレイと進捗バー付きダイアログを表示し、他の操作をできないようにする。
- 作成完了時はExplorerで生成された `.zip.gpg` ファイルを選択した状態で開く。

## 受け入れ条件

- 情報画面下部のボタン列から診断データ生成を実行できる。
- 生成中はオーバーレイが表示される。
- 生成成功後にExplorerで生成ファイルが選択される。
- 生成失敗時はエラーが表示され、オーバーレイが閉じる。
