# 042 Windows exe のプロパティ表示改善

## 状態

完了

## 問題

Windowsで `ClipForVRChat.exe` のプロパティを開くと、ファイルバージョンや製品バージョンがWails既定値の `1.0.0` として表示される。著作権も既定値のような表示になっており、配布物として分かりにくい。

## 期待する挙動

Windowsのファイルプロパティで、ClipForVRChatの製品名、説明、著作権、バージョンが適切に表示される。
表示上のファイル説明、ファイルバージョン、製品バージョンには `ClipForVRChat v0.1.5.0c112a8` のように、アプリ名、タグ、コミットIDを含む完全な識別情報をできるだけ表示する。

## 受け入れ条件

- ローカルWindowsビルドでexeのファイルバージョンが現在のタグに基づく数値バージョンになる。
- 製品バージョンも同じ数値バージョンになる。
- 著作権が `Copyright (c) 2026 hatolife` になる。
- 製品名が `ClipForVRChat` として表示される。
- ファイル説明、ファイルバージョン、製品バージョンの文字列に `ClipForVRChat vX.Y.Z.<commit>` 形式の情報が表示される。
- GitHub ActionsのWindows Releaseビルドでも同じ情報を埋め込める。
