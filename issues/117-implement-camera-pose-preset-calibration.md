# 自動撮影の現在Pose保存と構図管理を実装する

## 問題

v0.1.8-rc5 時点では、初期構図が未キャリブレーションのテンプレートのまま撮影に使われ、実際の「正面」「背後」「斜め」構図を保存できない。
仕様書ではMVP必須として、VRChatから `/usercamera/Pose` を受信し、現在のゲーム内カメラPoseを構図プリセットへ保存できることが求められている。

## 期待する挙動

VRChat内でUser Cameraを配置して少し動かすと、アプリが最新Poseを受信し、選択した構図へ保存できる。
v0.1.8の最終仕様ではIssue 120の通り、未キャリブレーションかどうかと撮影対象かどうかを分離し、撮影対象は構図ごとの「撮影する」トグルで決める。

## 受け入れ条件

- アプリがVRChatからのOSC受信ポートを開き、`/usercamera/Pose` を最新Poseとして保持する。
- 最新Poseの鮮度が設定値以内の場合、既存構図へ上書き保存できる。
- 最新Poseの鮮度が設定値以内の場合、新規構図として追加できる。
- 構図の追加、複製、削除、並び替え、有効/無効、名前変更が設定画面からできる。
- 現在Pose保存済みかどうかを設定画面で確認できる。
- 既存の自動投稿、Discord投稿、履歴、診断ログを壊さない。

## v0.1.8 最小要件チェックリスト

- [x] 必要な設定値が設定画面の「自動撮影」タブから設定できる。
- [x] 初期値に従って、VRChat内のUser Cameraを使った自動撮影が動作する。
- [x] 解像度設定は難しい場合のみ断念し、断念した場合はUI/ログ/Issueで明示する。
- [x] 撮影時点でインスタンスにいたユーザー情報をVRChat output logから取得する。
- [x] 取得したユーザー情報を画像と紐づくsidecar JSONへ保持する。
- [x] Discord投稿本文に、設定に従って同席ユーザー情報を含める。
- [x] EXIFへユーザー情報を入れない場合は、sidecar JSONを正本として明示し、設定名で誤解を生まない。
- [x] 初期3構図は妥当なPoseと拡大率を持ち、現在Pose保存で任意に上書きできる。
- [x] 未キャリブレーションかどうかと撮影トグルを分離し、初期Poseでも撮影できる。
- [x] 既存機能の自動投稿、通常画像処理、履歴、診断ログが壊れていない。
