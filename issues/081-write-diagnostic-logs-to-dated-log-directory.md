# 081 診断ログを日付付きログフォルダへ出力する

## 問題

診断ログが `config.json` と同じフォルダの `diagnostic.log` に1本で出力されている。
不具合調査用の診断パッケージではログフォルダを含める想定のため、ログ出力先がフォルダ構成になっていないと扱いづらい。

## 期待する挙動

- 診断ログは `config.json` と同じフォルダを基準に `log/YYYY-MM-DD.log` へ出力する。
- 空の設定パスではカレントディレクトリ基準の `log/YYYY-MM-DD.log` を使う。
- 診断パッケージには `log/` 配下のログファイルを含める。

## 受け入れ条件

- 操作ログ、処理ログ、起動ログが `log/YYYY-MM-DD.log` に出力される。
- 診断パッケージ作成対象が `log/` 配下のログファイルを収集する。
- 既存テストが日付付きログ出力先に追従している。
