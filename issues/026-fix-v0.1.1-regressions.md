# v0.1.1 不具合修正

## 対象

- 結果が空のときに履歴画面を開けない
- 履歴画面への導線がクリアボタン長押しでわかりにくい
- Win+Shift+S のクリップボード画像を処理すると、左端に右端の列が回り込むことがある

## 調査メモ

Win+Shift+S の例では、処理後画像が単なるリサイズや再エンコードではなく横方向に数ピクセルずれていた。
リサイズ不要な小さい画像でも発生するため、画像縮小処理ではなく Windows クリップボードから PNG 化する段階の DIB 変換が原因の可能性が高い。

## 対応方針

- Windows ではクリップボードに登録 PNG 形式が存在する場合、それを優先して読み取る
- 登録 PNG が取れない場合のみ従来の `golang.design/x/clipboard` 経路にフォールバックする
- 履歴画面は結果や履歴が空でも開ける
- クリアとは別に履歴画面への導線を用意する

## 完了条件

- Win+Shift+S の小さいクリップボード画像で端の回り込みが発生しない
- 結果が空でも履歴画面に遷移できる
- クリア操作と履歴画面への遷移が分かれている
