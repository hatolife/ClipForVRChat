# 設定画面・結果画面・情報画面のUI改善

## 問題

設定画面のレイアウトが崩れていて見づらい。結果画面もURL文字列が目立ちすぎており、サムネイル中心の表示にしたい。情報画面からGitHubやOSSライセンス情報へ遷移できる必要がある。

## 期待する挙動

- 設定画面を見栄えよく再設計する。
- 出力拡張子を設定できるようにする。既定値はPNGで、JPGも選べる。
- JPEG品質はJPG出力時にだけ適切に表示または有効化する。
- 設定保存後は設定画面を閉じて初期画面へ戻る。
- 設定画面下部に閉じるボタンを置く。
- 情報画面のGitHub URLはクリックでブラウザを開く。
- 情報画面からOSSライセンス表記画面へ遷移できる。
- OSSライセンス画面には使用OSS一覧とライセンス表記を見やすく表示する。
- 結果表示はサムネイル画像中心にし、クリックでURLを再コピーできることを明記する。
- 結果が大量にある場合でも、スクロールは結果部分だけに限定し、左側のドラッグ&ドロップ領域は伸びない。
- 設定画面もスクロール時に操作しやすいレイアウトにする。

## 受け入れ条件

- 設定画面のチェックボックスとラベルが崩れず、視認しやすい。
- PNG/JPG出力設定が保存され、画像出力に反映される。
- PNG出力時にJPEG品質が不要であることがUI上分かる。
- GitHub URLクリックでブラウザが開く。
- OSSライセンス画面へ遷移できる。
- 結果画面は大きめのサムネイルカードを表示し、URL文字列は主表示にしない。
- 結果一覧が多い場合、ドロップ領域は画面内に収まり、結果一覧だけが縦スクロールする。
- 設定画面の入力欄、チェック項目、保存/閉じるボタンが画面幅に応じて自然に並ぶ。
