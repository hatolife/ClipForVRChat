# 設定・履歴ファイルの権限と排他制御

## 問題

Webhook URL や Discord 削除 token を含む `config.json` / `history.json` が広めの権限で保存される。また自動投稿と手動操作が同時に履歴ファイルと UI 状態を更新すると、履歴の欠落やフラグ消失が起きる可能性がある。

## 期待する挙動

設定・履歴ファイルは可能な範囲でユーザーだけが読める権限で保存され、履歴更新と UI 状態更新は同時実行時にも破損しない。

## 受け入れ条件

- `config.json` と `history.json` は `0600` 相当で保存される。
- 設定・履歴ディレクトリは `0700` 相当で作成される。
- `App` の履歴更新と state 更新が mutex で直列化される。
- 履歴更新の単体テストまたは既存テストで破損しないことを確認する。
