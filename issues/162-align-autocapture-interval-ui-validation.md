# 自動撮影間隔のUI最小値をNormalizeと一致させる

## 問題

自動撮影タブの「撮影間隔」はUI上 `min=1` で入力できる。
一方、`AutoCaptureConfig.Normalize()` は `captureIntervalSec < 10` を10秒へ丸める。
ユーザーが1-9秒を設定できるように見えるが、保存後や実行時には10秒へ変わる。

## 期待する挙動

UI、Normalize、説明文の最小値が一致し、ユーザーが設定した値と実際のスケジューラ挙動が食い違わない。

## 受け入れ条件

- v0.1.8で許可する最小撮影間隔を決める。
- 最小10秒を維持する場合は、UIの `min`、説明、バリデーションを10秒に合わせる。
- 1秒以上を許可する場合は、Normalizeとスケジューラ負荷/重複制御を1秒対応へ変更する。
- 保存後に画面上の値が実際に保存された値へ同期される。
- 境界値の単体テストまたはフロント入力検証を追加する。
